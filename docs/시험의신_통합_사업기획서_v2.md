## 1. Executive Summary

> **핵심 요약**
>
> **시험의 신**은 학생이 자신의 시험지를 업로드하면 AI가 출제 패턴 분석(블루프린트), 오답 원인 진단, 교정 변형문항 생성까지 자동으로 수행하는 **내신 대비 플랫폼**입니다.
>
> **v2.0 주요 변경사항:**
> - OCR 엔진을 Upstage에서 **GPT-4o-mini Vision**으로 변경 → API 비용 절감 + 파이프라인 단순화
> - **모바일/태블릿 직접 풀이 캔버스** 추가 → 생성된 문제를 기기에서 바로 풀 수 있는 인터랙티브 환경
> - **학부모 연동 대시보드** 추가 → 자녀 계정 연동 후 학습 성과/취약점 실시간 확인
> - **소셜 학습 기능** 추가 → 팔로우/팔로잉 기반 문제 공유 및 오답노트 공유
>
> MVP는 중고등 수학 내신에 특화하여 **8주(약 110시간)** 내 출시하며, 문항 크레딧 기반 원가연동 과금으로 수익성을 확보합니다.

### v1.0 → v2.0 변경 요약

| 영역 | v1.0 | v2.0 | 변경 이유 |
|------|------|------|-----------|
| OCR 엔진 | Upstage Document Parse ($0.02/page) | GPT-4o-mini Vision (~$0.005~0.01/page) | API 통합으로 비용 50~75% 절감, 파이프라인 단순화 |
| AI 파이프라인 | 7단계 (OCR → 분리 → 분류 → 해설 → 검산 → 오답 → 변형) | **6단계** (OCR+분리 통합 → 분류 → 해설 → 검산 → 오답 → 변형) | GPT-4o-mini Vision이 OCR과 문항 분리를 동시 수행 |
| 문제 풀이 | 결과물 확인만 가능 | **모바일/태블릿 직접 풀이 캔버스** | 학습 완결성 향상, 체류시간 증가 |
| 학부모 기능 | Phase 2 예정 | **MVP에 포함** (연동 대시보드) | 결제 결정권자(학부모) 조기 확보 |
| 소셜 기능 | 없음 | **팔로우/팔로잉 + 문제/오답노트 공유** | 바이럴 루프 강화, 리텐션 향상 |
| 개발 기간 | 6주 (79시간) | **8주 (110시간)** | 신규 기능 추가에 따른 현실적 조정 |

## 2. 시장 환경 및 사업성 타당성 분석

### 2.1 거시 환경: AI 디지털교과서 시대의 도래

2025~2026년은 대한민국 교육 산업의 기술적 변곡점입니다. 정부 주도 AI 디지털교과서(AIDT) 도입이 본격화되면서, AI 기반 기술 중심 스타트업에 전례 없는 기회를 제공합니다.

| 구분 | 지표 | 시사점 |
|------|------|--------|
| 전체 시장 규모 | 2026년 약 11조 원 (PwC 추정) | 충분한 시장 규모로 진입 타당성 확보 |
| 정책적 변수 | AI 디지털교과서 초·중·고 단계적 도입 | 학생/학부모의 AI 도구 수용성 급증 |
| 교사 인식 | AIDT 사교육 경감 효과 부정적 89.2% | 공교육 AI가 채우지 못하는 '시험 대비' 니즈 건재 |
| 소비자 심리 | AI 기술 효율성 신뢰도 57% 이상 | AI 기반 학습 도구에 대한 심리적 진입장벽 낮음 |
| 기술 인프라 | Claude 4.5/4.6, GPT-4o 등 초고성능 LLM 상용화 | 1인 개발자도 전문가 수준 서비스 구현 가능 |

### 2.2 TAM-SAM-SOM 시장 규모 추정

| 구분 | 대상 | 추정 규모 | 산출 근거 |
|------|------|-----------|-----------|
| TAM | 국내 에듀테크 전체 | 11조 원 | PwC 2026 전망치 |
| SAM | 중고등 내신 AI 학습 도구 | 약 5,000억 원 | 에듀테크 시장의 약 4.5% |
| SOM | 수학 내신 AI 분석 플랫폼 | 약 150~300억 원 | SAM의 3~6% |

### 2.3 사업성 타당성 종합 판단

> **사업성 타당성 결론: GO**
>
> - **수익성**: 유료 전환율 3~7% 가정 시, 6개월 내 흑자 전환 가능. 문항 크레딧 과금으로 AI 원가 변동에 안전한 마진 구조 확보.
> - **확장성**: 중고등 수학 → 과학 → 전과목 → 대학/자격증까지 수평적 확장 용이. B2B 학원 라이선스로 수익 다각화.
> - **개발 효율**: 바이브코딩 적용으로 1인 개발 110시간 MVP 가능. 최신 AI 모델 출시 당일 즉시 적용 가능한 기동성.
> - **기술 경쟁력**: GPT-4o-mini Vision OCR + Claude 4.5 Sonnet 조합으로 수식 인식 및 STEM 추론 최고 수준 확보.
> - **진입 장벽**: 사용자 업로드 기반 OSP 모델로 콘텐츠 확보 비용 제로. 저작권 리스크 최소화 설계.

### 3.2 차별화 포지셔닝

> **핵심 차별화 5줄 요약**
>
> 1. 콴다는 '문제 1개'를 풀어준다 → 시험의 신은 **'시험지 1장'을 플랜으로 변환**한다
> 2. 족보닷컴은 기출문제를 '보여준다' → 시험의 신은 내 오답을 **'교정'해준다**
> 3. 산타토익은 '점수를 예측'한다 → 시험의 신은 **'점수를 올려'준다**
> 4. 모든 경쟁사는 '결과만 보여준다' → 시험의 신은 **모바일에서 직접 풀게 한다**
> 5. 혼자 공부한다 → 시험의 신은 **친구와 오답노트를 공유하며 같이 공부한다**

### 4.1 타겟 고객 페르소나

| **학원 원장 박 씨** | 중소형 학원 수학 전담 | 학생별 맞춤 교재 부족, 시간 부족 | B2B 대시보드 + 학원명 PDF 예측 문제지 (Phase 2) |

### 4.2 고객 여정 및 Pain Point 매핑

| 여정 단계 | 학생의 행동 | Pain Point | 시험의 신 솔루션 |
|----------|------------|-----------|-----------------|
| 시험 후 (D+1) | 시험지 받아서 채점 확인 | 어디서 왜 틀렸는지 모름 | 시험지 업로드 → 오답 원인 3유형 분류 |
| 오답 분석 (D+2~3) | 틀린 문제 다시 풀어봄 | 같은 문제만 다시 풀면 실력 안 늘음 | 교정 변형문항 10~20개 자동 생성 + **모바일에서 직접 풀기** |
| 친구와 공유 (D+3~5) | 카톡으로 시험지 사진 공유 | 공유해도 분석 없이 사진만 | **★ 오답노트/변형문항 공유 → 친구도 바로 풀기** |
| 학부모 확인 (D+1~7) | 자녀에게 "시험 어땠어?" 질문 | 구체적 학습 상황 파악 불가 | **★ 학부모 대시보드 자동 알림** |
| 다음 시험 대비 (D-14) | 문제집 구매, 학원 추가 | 어떤 단원을 집중해야 할지 모름 | 블루프린트로 출제 비중 시각화 |
| 직전 대비 (D-3~1) | 벼락치기, 불안감 극대화 | 효율적 복습 방법 부재 | 10분 교정 루틴 + **미니테스트 모바일 풀이** |

### 4.3 고객 니즈 검증 방법론

> **PMF 검증을 위한 고객 니즈 확인 계획**
>
> - **1차 검증 (MVP 전)**: 학생 커뮤니티(수만휘, 오르비) 설문 100명 + 심층 인터뷰 10명 → 시험지 업로드 의향, **모바일 풀이 니즈**, **친구 공유 의향**, WTP(지불 의향 가격) 확인
> - **2차 검증 (베타)**: 50명 베타 테스터 → Activation Rate 60% 이상 확인 (업로드 → 오답 3개 이상 체크 → **모바일 풀이 1회 이상**)
> - **3차 검증 (런칭 후)**: 시험 시즌 Retention Rate 40% 이상 + 미니테스트 정답률 +15%p + **소셜 공유율 20% 이상** 확인
> - **정성 지표**: '이 앱 덕분에 시험 점수가 바뀌었다'는 사용자 증언 수집 → 바이럴 마케팅 소재 활용

## 5. 핵심 기능 상세 설계

### 5.1 MVP 범위: 중고등 수학 내신 특화 + 모바일 풀이 + 소셜

> **수학 특화 MVP 선택 이유 4가지**
>
> 1. **검증 용이성**: 수식/계산 결과를 2중 검증(모델A 풀이 + 모델B 검산)할 수 있어 신뢰도 확보 가능
> 2. **오답 분류 명확성**: '개념 부족 vs 계산 실수 vs 시간 부족' 분류가 비교적 명확
> 3. **변형문항 품질**: 수치 변경, 조건 변환 등 정확한 변형이 가능하여 AI 생성 품질 높음
> 4. **환불/이탈 리스크 최소**: 정답이 명확한 과목이므로 '해설이 틀렸다' 클레임 방어 용이

### 5.2 사용자 플로우 (6단계 — v2.0 통합 파이프라인)

| Step | 사용자 행동 | AI 처리 | 결과물 |
|------|-----------|---------|--------|
| 1 | 시험지 사진/PDF 업로드 | **GPT-4o-mini Vision: OCR + 문항 분리 동시 수행** | 구조화된 문항 JSON (번호, 본문, 보기, 정답, 배점) |
| 2 | 문항 분리 확인 (UI) | 사용자 검증 + 수정 | 문항별 카드 (사용자 검증 완료) |
| 3 | - | 단원/유형/난이도/배점 추정 | ★ 시험지 블루프린트 |
| 4 | 틀린 문항 체크 / 답안 입력 | 오답 원인 분해 (3유형) + 해설 생성 + 2중 검증 | ★ 오답 진단 카드 + 해설 |
| 5 | - | 교정 변형문항 10~20개 생성 | ★ 교정 변형문항 세트 |
| 6 | **모바일/태블릿에서 미니테스트 직접 풀이** | 채점 + 점수 예측 | 점수 예측 + 다음 학습 추천 + **친구 공유 옵션** |

> **v1.0 대비 변경**: Step 1에서 Upstage OCR → GPT-4o-mini Vision 호출 1회로 OCR + 문항 분리를 동시 수행. 파이프라인이 7단계에서 **6단계로 단순화**.

### 5.3 핵심 기능 ①: 시험지 블루프린트

업로드된 시험지에서 '단원/유형/난이도/배점'을 구조화하여 시각적 문제 분포 맵을 생성합니다. 여러 시험지 누적 시 해당 학교/학년의 출제 비중을 개인 데이터만으로 추정할 수 있습니다.

> **블루프린트 출력 예시**
>
> 과목: 수학 | 학년: 고등2 | 시험: 1학기 중간
>
> - 함수 단원: 8문항(40%) | 확률과통계: 5문항(25%) | 미적분: 4문항(20%) | 수열: 3문항(15%)
> - 문제유형: 객관식 15개(75%) + 주관식 3개(15%) + 서술형 2개(10%)
> - 난이도 분포: 쉬움 30% | 보통 45% | 어려움 25%
> - → 인사이트: '함수 단원 비중 높음, 서술형 문제에서 난이도 높아지는 패턴'

### 5.4 핵심 기능 ②: 오답 원인 분해 (3유형 분류)

| 오답 유형 | AI 판단 기준 | 교정 방법 |
|----------|-------------|-----------|
| 📖 개념 부족 | 해당 단원 기초 문제도 틀림 / 풀이 과정에서 근본 공식 오용 | 개념 설명 카드 + 기초 변형문항 5개 (1분 풀이 난이도) |
| ✍️ 계산 실수 | 풀이 방향은 맞지만 중간 계산 오류 / 부호 실수 | 동일 구조 수치변형 문항 5개 (실수 유발 지점 강조) |
| ⏱️ 시간 부족 | 같은 유형 쉬운 문제는 맞힘 / 어려운 문제만 틀림 | 시간제한 미니테스트 (5문제/10분) + 빠른 풀이 팁 |

### 5.5 핵심 기능 ③: 교정 변형문항 엔진

콴다의 '유사문항'과 결정적으로 다른 점: **'내 오답 원인을 교정하는 변형문항'**을 생성합니다.

| 오답 유형 | 변형문항 전략 | 예시 |
|----------|-------------|------|
| 개념 부족 | 같은 개념을 다른 맥락에서 묻는 문제 | 이차함수 → 실생활 문제에서 이차함수 찾기 |
| 계산 실수 | 같은 구조에서 수치만 변경 + 함정 유발 보기 | 부호실수 유발 수치: -3 → -5로 변경 |
| 시간 부족 | 빠른 풀이법 훈련용 변형 | 복잡한 전개 대신 짧은 지름길 팁 제공 |

### 5.6 ★ 핵심 기능 ④: 모바일/태블릿 직접 풀이 캔버스 (신규)

> **"생성된 문제를 보기만 하는 게 아니라, 기기에서 직접 풀 수 있다"**

AI가 생성한 교정 변형문항과 미니테스트를 모바일/태블릿에서 **인터랙티브하게 직접 풀이**할 수 있는 환경을 제공합니다. 이는 기존 에듀테크 서비스에서 '결과만 보여주는' 한계를 넘어, **학습의 완결성**을 플랫폼 안에서 달성합니다.

#### 5.6.1 풀이 모드 상세

| 풀이 모드 | 설명 | UI 요소 |
|----------|------|---------|
| **객관식 터치 풀이** | 5지선다 보기를 터치/클릭으로 선택 | 보기 카드 5개 + 정답 확인 버튼 |
| **주관식 입력 풀이** | 숫자/수식 키패드로 답 입력 | 수학 키패드 (분수, 루트, 지수 지원) |
| **서술형 풀이 캔버스** | 태블릿 펜/손가락으로 풀이 과정 작성 | Canvas 드로잉 + 이미지 저장 (Phase 2: AI 채점) |
| **타이머 미니테스트** | 시간제한 + 문제 세트 연속 풀이 | 상단 타이머 바 + 문제 번호 네비게이션 |

#### 5.6.2 인터랙티브 풀이 UX 흐름

```
[변형문항 세트 생성] → [풀이 모드 선택] → [문제별 풀이]
     ↓                                         ↓
[타이머 시작 (선택)]                    [정답 제출]
                                              ↓
                                    [즉시 채점 + 해설 표시]
                                              ↓
                                    [전체 결과 + 점수]
                                              ↓
                                    [친구에게 공유] or [오답노트에 추가]
```

#### 5.6.3 기술 구현 방안

| 요소 | 기술 | 비고 |
|------|------|------|
| 객관식/주관식 UI | React 컴포넌트 (Next.js) | 모바일 반응형 설계 |
| 수학 키패드 | 커스텀 React 키패드 컴포넌트 | 분수(½), 루트(√), 지수(x²), π 등 |
| 수식 렌더링 | KaTeX (경량 LaTeX 렌더러) | 변형문항의 수식을 아름답게 표시 |
| 서술형 캔버스 | HTML5 Canvas + Pointer Events API | 태블릿 펜 압력 감지 (Phase 2) |
| 타이머 | React 상태 관리 (zustand) | 브라우저 백그라운드 시에도 정확한 시간 관리 |
| 채점 엔진 | 클라이언트 사이드 (객관식) + 서버 사이드 (주관식 수식 비교) | 수식 정규화 후 비교 |
| PWA | next-pwa | 오프라인 캐싱으로 풀이 중 네트워크 끊김 대응 |

#### 5.6.4 모바일 최적화 요구사항

- **반응형 설계**: 320px(모바일) ~ 1024px(태블릿) ~ 1440px(PC) 전 범위 최적화
- **터치 타겟**: 최소 44x44px (iOS HIG 기준) 보기 카드
- **수식 가독성**: KaTeX 렌더링 폰트 크기 모바일에서 최소 16px 이상
- **오프라인 지원**: PWA 캐싱으로 풀이 중 네트워크 끊김 시에도 문제/보기 유지
- **풀이 상태 자동 저장**: 풀이 중 앱 종료 → 재진입 시 이어 풀기 (localStorage + Supabase 동기화)

### 5.7 ★ 핵심 기능 ⑤: 학부모 연동 대시보드 (신규)

> **"결제 결정권자(학부모)에게 가치를 직접 보여주는 핵심 전환 장치"**

학부모가 자녀 계정과 연동하여, 자녀의 학습 성과를 실시간으로 확인할 수 있는 전용 대시보드를 제공합니다.

#### 5.7.1 학부모-자녀 연동 흐름

```
[학부모 회원가입] → [자녀 연동 코드 발급 요청]
                           ↓
              [자녀가 자신의 계정에서 연동 코드 입력 + 동의]
                           ↓
              [연동 완료 → 학부모 대시보드 활성화]
```

- **연동 방식**: 자녀가 생성한 6자리 연동 코드를 학부모가 입력하는 방식 (자녀의 명시적 동의 필수)
- **다자녀 지원**: 학부모 1명이 최대 5명의 자녀 계정 연동 가능
- **연동 해제**: 자녀가 언제든 연동 해제 가능 (자녀 주도권 보장)

#### 5.7.2 학부모 대시보드 구성

| 대시보드 섹션 | 표시 정보 | 업데이트 주기 |
|-------------|----------|-------------|
| **학습 현황 요약** | 이번 주 풀이 문항 수, 학습 시간, 접속일 수 | 실시간 |
| **취약점 히트맵** | 단원별 정답률 히트맵 (붉은색 = 취약) | 시험지 분석마다 갱신 |
| **점수 추이 그래프** | 미니테스트 점수 추이 (시간축 그래프) | 미니테스트 완료 시 |
| **오답 패턴 리포트** | '개념 부족 vs 계산 실수 vs 시간 부족' 비율 파이차트 | 주간 자동 리포트 |
| **시험 일정** | 다음 시험까지 D-day + 추천 학습 플랜 | 수동 설정 |
| **활동 알림** | 자녀가 시험지 업로드/미니테스트 완료 시 알림 | 이벤트 기반 |

#### 5.7.3 학부모 알림 시스템

| 알림 유형 | 조건 | 채널 |
|----------|------|------|
| 학습 완료 알림 | 자녀가 미니테스트 완료 시 | 인앱 + 이메일 (선택) |
| 주간 리포트 | 매주 일요일 자동 발송 | 이메일 |
| 취약점 경고 | 특정 단원 정답률 50% 미만 시 | 인앱 알림 |
| 학습 장기 미접속 | 7일 이상 미접속 시 | 이메일 |

#### 5.7.4 프라이버시 고려사항

- 학부모에게 **노출되는 정보**: 학습 통계, 점수, 취약점 (집계 데이터)
- 학부모에게 **노출되지 않는 정보**: 시험지 원본, 개별 문항 답안, 소셜 활동 내역
- **자녀 동의 필수**: 연동 코드를 자녀가 직접 발급하고, 학부모에게 전달
- **연동 해제 자유**: 자녀가 설정에서 즉시 연동 해제 가능

### 5.8 ★ 핵심 기능 ⑥: 소셜 학습 — 팔로우/팔로잉 & 공유 (신규)

> **"혼자 공부하던 내신 대비를 함께 공부하는 경험으로 전환"**

같은 학교/학년 친구를 팔로우하고, 문제 세트와 오답노트를 공유하는 소셜 학습 기능입니다.

#### 5.8.1 소셜 시스템 구조

```
[사용자 A] ──팔로우──→ [사용자 B]
     │                      │
     │  ┌──────────────────┐│
     │  │  공유 가능 콘텐츠  ││
     │  │                  ││
     │  │  · 변형문항 세트  ││
     │  │  · 오답노트       ││
     │  │  · 미니테스트 결과 ││
     │  │  · 블루프린트     ││
     │  └──────────────────┘│
     │                      │
     └───── 피드에서 확인 ───┘
```

#### 5.8.2 팔로우/팔로잉 시스템

| 기능 | 설명 | 비고 |
|------|------|------|
| **사용자 검색** | 닉네임으로 검색 (실명 비노출) | 개인정보 보호 |
| **팔로우 요청** | 팔로우 시 상대방 수락 필요 (비공개 기본) | 청소년 안전 |
| **팔로잉 피드** | 팔로우한 친구가 공유한 콘텐츠 피드 | 시간순 정렬 |
| **팔로워/팔로잉 수** | 프로필에 표시 | 최대 200명 제한 (스팸 방지) |

#### 5.8.3 공유 가능 콘텐츠 및 규칙

| 공유 콘텐츠 | 공유 형태 | 개인정보 보호 |
|-----------|----------|-------------|
| **변형문항 세트** | AI가 생성한 변형문항 (원본 시험지 미포함) | ✅ 원본 시험지는 절대 공유 불가 |
| **오답노트** | 내가 틀린 유형 + 교정 문항 묶음 | ✅ 틀린 답안/점수는 비공개 선택 가능 |
| **미니테스트 결과** | 점수 + 정답률 (상세 답안 제외) | ✅ 공유 여부 선택 |
| **블루프린트** | 시험 출제 비중 분석 | ✅ 학교명 비노출 옵션 |

> **⚠️ 저작권 보호 핵심 원칙**: 사용자가 업로드한 **원본 시험지는 절대 공유할 수 없습니다**. 공유 가능한 것은 **AI가 새로 생성한 콘텐츠**(변형문항, 오답노트, 블루프린트)뿐입니다. 이는 OSP 면책 원칙을 소셜 기능에서도 유지하는 핵심 설계입니다.

#### 5.8.4 소셜 학습 시나리오

**시나리오 1: 시험 후 오답노트 교환**
```
민지: 중간고사 오답노트 공유 → "함수 단원 개념 부족 유형 교정 문항 5개"
준혁: 피드에서 확인 → "나도 이 유형 약했는데!" → 바로 모바일에서 풀기
```

**시나리오 2: 기말고사 대비 문제 공유**
```
준혁: 미적분 변형문항 세트 생성 → 친구 3명에게 공유
친구들: 각자 모바일에서 풀이 → 결과 비교 → 자연스러운 경쟁/동기부여
```

**시나리오 3: 블루프린트 공유로 출제 예측**
```
민지: "우리 학교 수학 선생님 함수 40% 출제" 블루프린트 공유
같은 학교 학생: 팔로우 → 내 시험 대비에 참고
```

#### 5.8.5 바이럴 루프 강화 효과

```
① 학생이 시험지 업로드 → 오답 분석 + 변형문항 생성
② 모바일에서 직접 풀기 → 점수 확인
③ "야 나 이 유형에서 점수 올랐어!" → 오답노트 친구에게 공유
④ 친구가 공유받은 문항을 바로 모바일에서 풀기
⑤ 친구도 자기 시험지 업로드 → 순환 ↻
```

### 9.3 바이럴 루프 (v2.0 강화)

```
① 학생이 수학 시험지 찍어 업로드
② 30초 만에 블루프린트 + 오답진단 + 교정 변형문항 → 'WOW' 순간
③ 모바일에서 변형문항 직접 풀기 → 점수 확인 → 성취감
④ "야 이거 진짜 내 실수까지 분석해준다" → 오답노트를 친구에게 공유
⑤ 친구가 공유받은 문항을 바로 모바일에서 풀기 → 시험의 신 가입
⑥ 친구도 자기 시험지 업로드 → 순환 ↻
⑦ 학부모가 대시보드에서 자녀 점수 상승 확인 → 주변 학부모에게 추천
```

> **v2.0 바이럴 강화 포인트**: 기존 '결과물 보여주기'에서 **'결과물 공유 + 직접 풀기'**로 진화. 소셜 공유가 단순 링크 공유가 아니라 **"친구가 바로 풀 수 있는 학습 콘텐츠 공유"**이므로 바이럴 전환율이 훨씬 높습니다.

## 12. 결론: 통합 전략의 승부처

> **핵심 승부처 7가지 (v2.0)**
>
> ① **내신 특화 루프**: 시험지 → 블루프린트 → 오답교정 → 변형문항 → 점수상승 (콴다/족보와 정면충돌 회피)
>
> ② **모바일 풀이 완결성**: 분석만 하는 게 아니라 **기기에서 직접 풀 수 있는** 학습 완결 경험
>
> ③ **소셜 바이럴**: 친구에게 오답노트/변형문항 공유 → **공유받은 친구가 바로 풀기** → 자연스러운 가입 유도
>
> ④ **학부모 Lock-in**: 결제 결정권자에게 자녀 학습 현황을 **실시간 대시보드**로 직접 보여주는 전환 장치
>
> ⑤ **신뢰 장치**: 2중검증 + 근거표시 + 확신도 → '예측 문제가 정말 맞나?' 의문 해소
>
> ⑥ **원가연동 과금**: 문항크레딧 + 시즌패스 + 학부모 플랜 → AI 비용 급증에도 마진 보전
