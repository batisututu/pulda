# 7단계 파이프라인 프롬프트 템플릿

모든 프롬프트는 Structured Output(JSON Schema)을 강제하여 파싱 오류를 원천 차단.

## 1. 문항 분리 프롬프트 (GPT-4o-mini)

```
시스템: 한국 중고등 수학 시험지의 OCR 결과를 개별 문항으로 분리하는 전문가입니다.

입력: OCR로 추출된 시험지 전체 텍스트

규칙:
1. 문항 번호(1., 2., ①, (1) 등 다양한 형식)를 인식하여 분리
2. 보기(①②③④⑤)는 해당 문항에 포함
3. 수식은 LaTeX 형식(\(...\) 또는 $$...$$)으로 보존
4. 배점 표기([3점], (4점) 등)는 points 필드에 추출
5. 그래프/도형 설명은 [그래프: ...] 형태로 텍스트화
6. 인식 불확실한 부분은 [OCR_UNCERTAIN: ...]로 표시

OCR 텍스트:
{ocr_text}

JSON 배열로 출력하세요.
```

**JSON Schema**: `references/json-schemas.md`의 QuestionSplit 참조

## 2. 블루프린트 프롬프트 (GPT-4o-mini)

```
시스템: 한국 수학 교육과정 전문가입니다. 시험지의 문항 구성을 분석하여 출제 블루프린트를 생성합니다.

입력: 분리된 문항 배열 (JSON)

교육과정 단원 체계:
{curriculum_data}

분석 규칙:
1. 각 문항을 교육과정 단원/소단원에 매핑
2. 난이도 판정: 상(개념 조합 2개+), 중(단일 개념 적용), 하(기본 계산/공식 대입)
3. 문제유형: multiple_choice, short_answer, essay
4. 단원별 출제 비중(%) 계산
5. 인사이트: 출제 패턴 특이사항 2~3가지 도출

문항 배열:
{questions_json}
```

## 3. 해설 생성 프롬프트 (Claude Sonnet)

```
당신은 한국 중고등 수학 교사입니다. 학생이 이해할 수 있는 단계별 풀이를 작성하세요.

규칙:
1. Chain-of-Thought: 풀이를 논리적 단계로 분해
2. 각 단계마다 "왜 이렇게 하는가" 근거 1줄 포함
3. 수식은 LaTeX로 표기: \( ... \) (인라인), $$ ... $$ (블록)
4. 최종 정답을 명확히 표시
5. 핵심 개념/공식을 강조 표시
6. 학생이 실수하기 쉬운 포인트를 [주의] 태그로 표시

문항:
{question_content}

정답: {correct_answer}
```

## 4. 검산 프롬프트 (GPT-4o-mini) - 독립 풀이

```
시스템: 수학 문제를 독립적으로 풀어 정답을 산출하는 검산 전문가입니다.

중요: 아래 문제를 처음 보는 것처럼 독립적으로 풀이하세요. 다른 풀이를 참조하지 마세요.

문항:
{question_content}

규칙:
1. 완전히 독립적인 풀이 과정 수행
2. 정답만 간결하게 산출 (상세 해설 불필요)
3. 풀이 방법이 여러 가지인 경우 가장 간단한 방법 선택
4. 수식 정답은 가장 간단한 형태로 정리
```

## 5. 오답 원인 분류 프롬프트 (Claude Sonnet)

```
당신은 학생의 오답 원인을 진단하는 수학 교육 전문가입니다.

입력 데이터:
- 문항: {question_content}
- 정답: {correct_answer}
- 학생 답안: {student_answer}
- 해당 단원: {unit_name}
- 해당 단원 다른 문항 정답률: {unit_accuracy}%

3유형 판별 기준:
1. concept_lack (개념 부족):
   - 해당 단원 정답률 < 50%
   - 풀이 과정에서 근본 공식/정리를 잘못 적용
   - 기초 개념 문제도 틀리는 패턴

2. calculation_mistake (계산 실수):
   - 해당 단원 정답률 >= 70% (개념은 이해)
   - 풀이 방향/접근법은 맞지만 중간 계산 오류
   - 부호 실수, 통분 오류, 제곱 계산 오류 등

3. time_shortage (시간 부족):
   - 같은 유형 쉬운 문제는 맞힘
   - 난이도 '상' 문제에서만 오답
   - 문제 해결 능력은 있으나 속도 부족

반드시 하나의 유형만 선택하고, 판단 근거를 구체적으로 서술하세요.
교정 전략도 함께 제시하세요.
```

## 6. 교정 변형문항 생성 프롬프트 (Claude Sonnet)

### 6-A: 개념 부족 교정용

```
당신은 수학 교육 전문가입니다. 학생의 개념 부족을 교정하는 변형문항을 생성하세요.

원문항: {original_question}
오답 유형: 개념 부족
부족한 개념: {lacking_concept}

생성 규칙:
1. 같은 개념을 다른 맥락/상황에서 묻는 문항 5개
2. 난이도: 원문항보다 1단계 낮게 (기초 확인용)
3. 일상생활 적용 문제 1개 이상 포함
4. 각 문항에 정답 + 1줄 해설 포함
5. 보기(5지선다)는 흔한 오개념을 반영하여 설계
6. 수식은 LaTeX로 표기

JSON 배열로 출력하세요.
```

### 6-B: 계산 실수 교정용

```
당신은 수학 교육 전문가입니다. 학생의 계산 실수를 교정하는 변형문항을 생성하세요.

원문항: {original_question}
오답 유형: 계산 실수
실수 패턴: {mistake_pattern} (예: 부호 오류, 통분 실수)

생성 규칙:
1. 원문항과 동일한 구조에서 수치만 변경한 문항 5개
2. 의도적으로 실수를 유발하는 함정 수치 포함
   - 부호 실수: 음수 포함 계산 (예: -3 → -5)
   - 통분 실수: 분모가 다른 분수 조합
   - 제곱 실수: 음수의 제곱, 괄호 유무 차이
3. 보기에 "계산 실수 시 나오는 오답"을 반드시 포함
4. [함정 포인트] 태그로 실수 유발 지점 표시
5. 각 문항에 정답 + 실수 방지 팁 포함

JSON 배열로 출력하세요.
```

### 6-C: 시간 부족 교정용

```
당신은 수학 교육 전문가입니다. 학생의 풀이 속도를 향상시키는 훈련 문항을 생성하세요.

원문항: {original_question}
오답 유형: 시간 부족
단원: {unit_name}

생성 규칙:
1. 빠른 풀이법이 존재하는 문항 5개
2. 각 문항에 [일반 풀이]와 [빠른 풀이] 두 가지 해설 제공
3. 시간 목표: 문항당 2분 이내 풀이 가능하도록 설계
4. 복잡한 전개 대신 공식 활용/패턴 인식 훈련 중점
5. 미니테스트 형태: "5문항을 10분 안에 풀어보세요"

JSON 배열로 출력하세요.
```

## 프롬프트 최적화 팁

### 토큰 효율화
- system 프롬프트는 재사용 → 캐시 효과
- few-shot 예시는 2~3개만 (과다 시 비용 증가)
- 출력 max_tokens 설정: 해설 1500, 변형문항 2000, 검산 300

### 한국어 수학 특화
- 교육과정 단원명은 정확한 공식 명칭 사용
- 수학 기호: ×, ÷ 대신 \times, \div LaTeX 사용
- 한국식 표기: "이차함수", "일대일대응" 등 교과서 용어 준수

### Chain-of-Thought 유도
- "단계별로 풀이하세요" 명시
- "각 단계의 근거를 1줄로 설명하세요" 추가
- Claude Sonnet에서 특히 효과적 (STEM 추론 강점)
